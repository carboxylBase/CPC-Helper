# 起手式

现在我需要你对一个 Rust CLI 项目进行维护, 下面我会给你一个本项目的总结性文档, 你需要根据文档中的提示, 对我接下来交给你的任务进行处理.

此外, 每次你提交代码时, 都需要指明被修改项文件路径, 并且你需要将这个文件被修改后的完整代码发给我, 不能只发送被修改的部分, 防止引起交流上的误会.

如果这个任务需要多个步骤去完成, 你应当尽可能将这个任务分解为足够原子化的步骤, 保证操作的安全性.

如果在编写代码之前, 你需要了解某些文件原先的内容, 可以向我发出提问.

切记如果你打算修改某些文件, 又担心会影响他原本的功能, 一定要先向我索取源文件进行查看, 再编写代码.

如果你需要获取一些调试信息来编程, 可以在代码中添加获取调试信息的功能, 并且向我索取调试信息.

# 布置任务

我需要你补充???功能, 首先, 你需要提供若干可行方案供我选择, 由我作出决定后才进行具体的代码.

# 结束语

好了, 你的工作已经结束了, 合作愉快!

为了方便后面继续使用ai协助维护这个项目,  我需要你对这个项目进行一个概括性总结, 方便后来的ai快速上手这个项目, 并且知道这个项目的核心宗旨以及如何对本项目进行维护.

如果我给你发过上个版本的项目文档, 你需要在上个文档的基础上编写. 这个总结性文档应当包含以下内容:
+ 项目名称 (作为标题)
+ 项目概述 (Project Overview)
+ 完整的目录结构 (Directory Structure)
+ 核心设计与实现细节 (Core Design & Implementation)
+ 维护指南 (Maintenance Guide)
+ 关键代码片段备份 (Key Code Snippets)
+ 尚未解决的问题

你的回答应当只包含总结文档本身, 不需要其他内容.
